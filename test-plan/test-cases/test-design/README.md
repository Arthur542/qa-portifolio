# Test Design

## Overview

This section presents the test design techniques applied in this project, focusing on **Boundary Value Analysis (BVA)** and **Equivalence Partitioning (EP)**.

These techniques are used to ensure effective test coverage by validating input constraints, identifying edge cases, and reducing redundant test scenarios.

---

## Objective

The goal of this section is to:

* Apply structured test design techniques
* Validate system behavior based on input rules
* Identify both valid and invalid data conditions
* Improve test efficiency and coverage

---

## Techniques Applied

### 🔹 Boundary Value Analysis (BVA)

Used to validate system behavior at the limits of input ranges, focusing on minimum, maximum, and near-boundary values.

### 🔹 Equivalence Partitioning (EP)

Used to divide input data into valid and invalid groups, selecting representative values to optimize test coverage.

---

## Scope

The test design techniques were applied to:

* Username validation (6–30 characters)
* Password validation (8–30 characters)

Including:

* Valid inputs
* Invalid inputs
* Edge cases
* Boundary conditions

---

## Expected Outcome

* Ensure accurate validation of input fields
* Detect defects at boundary conditions
* Reduce redundant test cases
* Improve overall test effectiveness
