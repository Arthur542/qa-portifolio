# BDD (Behavior-Driven Development)

## Overview

This section contains the **BDD (Behavior-Driven Development) scenarios** for the project, written in Gherkin syntax to describe system behavior based on business rules.

---

## Objective

* Define system behavior in a clear and structured way
* Ensure alignment between business rules and expected outcomes
* Validate user flows from a behavioral perspective

---

## Content

This module includes:

* BDD scenarios written in Gherkin
* Business rule-based test scenarios
* Positive and negative flows
* Validation and conditional logic scenarios

---

## Scope

The scenarios cover:

* User Authentication
* Input validation (username and password rules)
* Promotion system behavior

---

## Expected Outcome

* Clear and readable test scenarios
* Validation of business rules through behavior
* Improved communication between QA and development

---
