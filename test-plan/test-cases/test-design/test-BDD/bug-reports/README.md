# Bug Reports

## Overview

This section contains the **bug reports** identified during the testing process, documenting defects found in the system based on executed test scenarios.

---

## Objective

* Identify and document system defects
* Provide clear and structured bug descriptions
* Support issue tracking and resolution
* Ensure software quality through defect analysis

---

## Content

This module includes bug reports related to:

* Calculation errors (e.g., subtotal issues)
* Input validation failures
* Incorrect handling of user data
* System behavior inconsistencies

---

## Scope

The reported bugs cover:

*  Subtotal calculation errors
*  Postal code validation issues
*  Name validation inconsistencies

Each bug report includes:

* Title
* Steps to reproduce
* Expected result
* Actual result
* Severity and priority
* Status

---

## Expected Outcome

* Clear identification of system defects
* Well-structured and reproducible bug reports
* Improved communication between QA and development
* Contribution to system reliability and quality
