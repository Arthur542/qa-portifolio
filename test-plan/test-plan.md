# Test Plan – QA Portfólio Project

## 1. Project Overview
This project contains manual test documentation created for practice and portfolio purposes.

The tests were executed using public test environments and simulated business rules to demonstrate different software testing techniques.

Some tests were performed on real websites, while others were based on hypothetical business rules to simulate real-world test scenarios.

----

## 2. Objectives
The objective of this project is to validate the system's functionality and demonstrate knowledge of manual testing techniques, including:

- Functional testing
- Positive testing
- Negative testing
- Boundary value analysis
- Equivalence partitioning
- Error reporting

----

## 3. Scope
The scope of the tests includes:

- Authentication and login validation
- Form validation
- Session control
- Validation of the checkout process
- Validation of input fields
- Validation of business rules

----

## 4. Testing Strategy
The testing strategy includes the manual execution of test cases designed to validate the system's behavior under valid and invalid conditions. The following techniques were used:

- Functional testing
- Boundary value analysis
- Equivalence partitioning
- Exploratory testing
